# ATS Resume Diagnostic (Open Skill)

Этот проект реализует стандарт **Open Skill** для глубокой диагностики резюме. Это не просто промпт, а полноценная методология анализа, которую может исполнить любой продвинутый AI-агент (Gemini CLI, Claude Code, Cursor и др.).

Скилл имитирует работу ATS-системы и опытного рекрутера, проверяя резюме на соответствие реальному рыночному спросу.

## Для чего это нужно
- Увидеть свое резюме глазами алгоритмов и "беглого взгляда" рекрутера.
- Понять, в какие "типажи" вакансий вы попадаете, а куда вход закрыт.
- Найти семантические разрывы (каких терминов и навыков не хватает в тексте).
- Проверить достижения на конкретику (XYZ-метрики).

## Как подготовить данные
Скилл ожидает, что в корне проекта или в папке `references/` лежат следующие файлы:

1.  **context.md** — ваш нарратив: желаемая роль, грейд, география, ограничения (например, только remote) и мотивация.
2.  **resume.pdf** (или .md / .txt) — актуальная версия вашего резюме.
3.  **1.md, 2.md ... 12.md** — тексты 12-20 вакансий, на которые вы хотите или планируете откликаться. 

> **Важно:** Для качественной диагностики нужно минимум 12 вакансий, чтобы AI мог выделить статистически значимые паттерны и "типажи" рынка.

## Как запустить

### 1. Через Gemini CLI (родная поддержка)
Если у вас установлен [Gemini CLI](https://github.com/google/gemini-cli):
```bash
# Установка
gemini skills install ./ats-resume-diagnostic.skill

# Запуск в чате
> "Проведи диагностику резюме по скиллу"
```

### 2. Через Claude Code / Cursor / OpenCode
Просто укажите агенту путь к файлу `SKILL.md` и попросите его выполнить инструкции:
> "Используй инструкции из SKILL.md для анализа файлов в папке references/"

### 3. Через NPM (для автоматизации)
```bash
npm install github:username/ats-resume-diagnostic
```

## Что вы получите на выходе
Результатом работы является **ATS Diagnostic Report**, который включает:
- **Scoring:** Процент подходимости к разным кластерам вакансий.
- **10-Second Recruiter:** Проверка "читаемости" грейда, домена и масштаба за первые 10 секунд.
- **Line-by-line Parsing:** Построчный разбор слабых мест в опыте.
- **Systemic Issues:** Анализ "глагольного переспама", отсутствия цифр или несоответствия тональности.

## Философия
Этот скилл следует принципам **Evidence Rule**: достижение считается существующим только при наличии измеримого результата. Он не переписывает резюме за вас, а ставит честный диагноз.
